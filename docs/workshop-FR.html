8. Programmation fonctionnelle : les bases de la programmation

8.1. Dfinition

La programmation fonctionnelle est un paradigme de programmation de type dclaratif qui considre le calcul en tant qu’valuation de fonctions mathmatiques.

Comme le changement d’tat et la mutation des donnes ne peuvent pas tre reprsents par des valuations de fonctions la programmation fonctionnelle ne les admet pas, au contraire elle met en avant l’application des fonctions, contrairement au modle de programmation imprative qui met en avant les changements d’tat.

source : Wikipedia

8.2. Immutabilit

Tout en Elm est immuable.

Cela signifie qu’une valeur ou une fonction ne peut pas tre modifie aprs sa cration.

{-- no keyword var, val or def
everything is immutable value, even the functions --}

myNumber = 42

increment x =
    x + 1

Vous ne modifiez jamais la valeur d’une variable, ce n’est pas possible.

Par exemple, l’ajout d’un lment en-tte d’une liste se fait en rutilisant l’ancienne liste. De faon sous-jacent, Le nouvel lment pointe simplement vers le premier lment de l’ancienne liste, qui n’est pas modifie.

8.3. Fonctions d’ordre suprieur

Les langages fonctionnels permettent l’utilisation de fonctions d’ordre suprieur.

Une fonction est dite d’ordre suprieur lorsqu’elle prend au moins une fonction comme paramtre, et/ou renvoie une fonction comme rsultat.

8.4. Fonctions pures

Une fonction est dite pure si elle ne produit aucun effet secondaire.

Le rsultat de l’appel une fonction pure dpend uniquement de la valeur de ses paramtres d’entre, et ne modifie rien d’autre que sa valeur de retour.

Cela prsente des avantages :

  • c’est facile tester

  • C’est souvent facile comprendre

  • c’est puissant en cas de multithreading (ce dont Elm ne tire pas parti, car il est bas sur JS)

Toutes les fonctions Elm sont pures. Cela signifie que :

  • elles ne sont bases sur aucun tat extrieur

  • elles ne produisent pas d’effets secondaires

  • elles doivent retourner un rsultat

C’est pourquoi, par exemple, en Elm, un "if" doit obligatoirement tre suivi d’un "else".

8.5. Fonctions de liste

L’immutabilit implique que vous ne pouvez pas faire une boucle "for" dans Elm : vous ne pouvez pas changer la valeur de l’index chaque itration.

Au lieu de cela, pourquoi vous utiliserez certaines fonctions applicables aux listes, typiques de la programmation fonctionnelle :

  • map pour appliquer une transformation chaque lment de la liste ;

  • filter pour extraire des lments de la liste correspondant un critre donn ;

  • fold (nomm "reduce" dans d’autres langages) pour construire une nouvelle structure de donnes partir de la liste.

8.6. Lambda

Lamba = fonction anonyme Lamba = fonction anonyme

Exemple :

List.map (\x -> x + 1) [1, 2, 3]

Un lambda est une fonction qui n’a pas de nom.

Dans l’exemple, la lambda est applique chaque lment de la liste, par la fonction map du module List. Elle ajoutera 1 chaque lment de la liste. x est la valeur de l’lment courant. Le rsultat sera une liste contenant[2, 3, 4].

En Elm, le caractre "antislash" est utilis pour introduire une fonction anonyme.
Il a t choisi parce que, visuellement, c’est le caractre le plus proche de la lettre grecque lambda.

9. Annotations de type

Les annotations de type sont trs utiles en Elm, bien qu’elles soient facultatives. En plus de documenter votre code, elles aident le compilateur fournir des messages utilisateurs clairs.

L’annotation de type dfinit le type de retour d’une fonction. C’est crit juste au-dessus du nom de la fonction, comme :

    x + 1

Une annotation de type en Elm comprend :

  • le nom de la fonction ou de la valeur ;

  • le symbole deux-points ;

  • une srie de types spars par des flches.

Le dernier type est le type de retour.

incrment : Int → Int signifie que la fonction increment prend un Int comme paramtre et retourne un Int.

Un nom de type dans Elm commence toujours par une lettre majuscule.

    a + b

Certains types sont paramtrs. Par l’exemple :

                              -- et retourne un Int

    ...

Un type peut-tre gnrique. Dans ce cas, le nom du type utilisera une ou plusieurs iquettes en minuscules. Par exemple, List a est une liste gnrique (similaire List<Any> ou List<T> dans d’autres langages).

List.length: List a -> Int -- prend une liste de n'importe quel type comme paramtre,
                           -- et renvoie un Int

    ...

Vous pouvez choisir n’importe quel nom pour les tiquettes, la seule contrainte est de commencer par une lettre minuscule. Il permet de donner une indication de ce que ce type doit contenir.

10. L’architecture Elm

L’architecture Elm est souvent appele "TEA" (The Elm Architecture)

10.1. Le pattern MVU

Image by Kolja Wilcke

Le principe du pattern MVU est le suivant :

  • l’application comprend un modle, une fonction de visualisation et une fonction de mise jour.

  • La fonction de visualisation est en charge du rendu du modle (elle va gnrer du Html)

  • La vue met des messages, par exemple pour indiquer que l’utilisateur a modifi le contenu d’un champ de saisie ou cliqu sur un bouton ;

  • La fonction update traite ces messages et met jour le modle en consquence.

  • Ce modle s’appelle MVU : Model - View - Update

10.1.1. Le pattern MVU par l’exemple

  1. le modle contient l’tat de l’application (appel store dans certains frameworks JS). Dans un premier temps il contient : deleteModal=False
    Ce modle est envoy la fonction view par le runtime Elm au dmarrage de l’application.

  2. la fonction view utilise le modle initial pour dcider d’afficher ou non la fentre de confirmation

  3. l’utilisateur clique sur le bouton "supprimer" de la page html.

  4. la vue met alors un message indiquant ce clic

  5. le runtime appelle la fonction update, et lui transmet le modle courant ainsi que le message envoy par la vue.

  6. la fonction update contient la logique appliquer pour chaque message

  7. pour DeleteClicked, la logique consiste stocker dans le modle la nouvelle valeur du champ deleteModal

  8. cette fin, la fonction update cre un nouveau modle, qui est une copie de l’ancien, mais dans lequel elle donne la valeur True au champ deleteModal.

  9. le runtime appelle la fonction view, et lui passe le modle dont elle doit gnrer le rendu.

  10. le nouveau modle contient deleteModal=True, donc la fonction de visualisation affiche la fentre modale permettant l’utilisateur de confirmer la suppression.

11.1. Effets de bord

  • Les effets de bord sont grs par le runtime

  • Les effets de bord sont invoqus via des commandes

  • Le rsultat d’un effet de bord est fourni via un message

Nous ne pouvons crire que des fonctions pures en Elm. Le rsultat de l’appel d’une fonction avec des valeurs de paramtres donnes retournera TOUJOURS la mme valeur, ce qui est pratique pour les tests.

Mais que faire si vous devez faire une opration avec un rsultat diffrent ? Par exemple, gnrer un nombre alatoire ? Ou faire une demande http ?

Dans ce cas, nous devrons mettre une commande qui demandera au runtime elm pour excuter la fonction impure. Le runtime fournira le rsultat de l’excution de cette fonction impure. en appelant la fonction update. Le message peut contenir, par exemple, le nombre alatoire gnr.

11.2. Architecture Elm avec effets de bord

Par rapport la version la plus simple de l’architecture elm, nous aurons des notions supplmentaires pour grer les traitements qui provoquent des effets de bord.

  1. la fonction update gnre un nouveau modle, et ventuellement une commande

  2. le runtime excute la commande. C’est l que tous les effets de bord sont traits (requte HTTP, gnration de nombres alatoires, etc.). Les commandes sont asynchrones.

  3. la fin de l’excution d’une commande, le runtime appelle la fonction update, et transmet au modle courant un message qui dcrit le rsultat de l’excution de la commande (succs ou chec, valeur, etc) ⇒ retour l’tape 1

11.9. Digression : le type de rsultat
